The problem

Every access change requires a physical visit to the guardhouse

Hikvision hardware is excellent. The management software is not. Resident enrollments, contractor passes, visitor pre-registration, move-out offboarding - HikCentral assumes you walk up to it every single time. For a modern estate or club, that is hours a week of admin time and a real security gap.

New resident or member fills in a paper form or PDF.
They make an appointment with the security person responsible for enrollment.
Both must physically meet at the guardhouse, same time, same place.
Security person manually types the details into the Hikvision system on the guardhouse computer.
A photo is taken right there using the Hikvision device.
Person is enrolled and they test it at the gate.

This requires a physical appointment, both parties present, around 5 minutes per person, the guardhouse computer to be accessible, and one person at a time. It becomes a total bottleneck. It is impossible on weekends or holidays unless someone staffs the guardhouse.

For a site going live with 2,000 people: 2,000 individual appointments. One at a time.

That is operational drag before the system starts delivering value.

Some estates have improved this slightly with digital forms sent via WhatsApp, but even those still require heavy manual work per person: downloading photos, copy-pasting data, assigning access levels, and checking whether the person is actually allowed access. Neither approach scales beyond a handful of people per week.

The solution

OnboardAI handles everything HikCentral doesn't

HikCentral is great for hardware and gate events. OnboardAI is great for everything else: onboarding, day-to-day access management, visitor flow, and reporting. Your existing Hikvision devices stay exactly as they are. We sit on top and add the operational layer.

Getting started

Five steps from setup to go-live

1

Deploy OnboardAI on a server

Provision a server, either a cloud VPS or an on-site machine at the site office. OnboardAI can be installed and running quickly.

2

Configure the access control connection

Install the lightweight on-site agent on a machine with network access to your access control system. Paste in the connection details from the admin panel. This is a once-off setup with your IT team.

3

Import your member list

Upload a CSV with your current member or resident list. OnboardAI uses this to validate new enrollments and match them against known people.

4

Share the onboarding link

Generate an enrollment link from the settings panel. Share it via WhatsApp broadcast, email, or print the QR code for reception. Members can start enrolling immediately.

5

Review and approve from the dashboard

Log in to the admin dashboard to see incoming enrollments. AI-verified submissions can be bulk-approved in one click, then queued for automatic transfer to the access control system.
